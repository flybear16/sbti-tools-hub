# SBTI Tools Hub — PRD

## 一、背景与目标

### 背景
- SBTI于2026年4月爆火，是现象级娱乐化人格测评
- 全网缺乏统一的测评收录平台，用户散落在各个复刻版之间
- 热点窗口预计3-6个月，需快速上线

### 目标
1. **短期**：2周内上线，截流SEO流量，快速变现
2. **中期**：成为SBTI及同类测评的权威导航站
3. **长期**：沉淀为娱乐测评分发入口

### 核心指标
- DAU > 1000（上线首周）
- Google SEO 关键词「SBTI」「SBTI测评」进入前3
- 广告收入覆盖运营成本

## 二、变现模式

### 主要收入
| 模式 | 说明 | 预期收益 |
|------|------|---------|
| **广告CPC** | 测评详情页插入Google AdSense/百度联盟 | 单次点击0.5-2元 |
| **CPS分佣** | 导流到测评工具，按完成次数分成 | 单次1-5元 |
| **推荐位** | 测评开发者付费求推荐 | 单次500-2000元/月 |
| **软文** | 测评推荐文章，植入品牌合作 | 单篇2000-5000元 |

### 次要收入
- 付费投稿：测评工具提交收录，收取listing费
- 私域导流：引导关注公众号/加群，后续变现

## 三、用户价值

**目标用户**：想测SBTI、测完还想测更多同类测评的用户

**用户需求**：
1. 找一个能测的SBTI工具（当前分散在各复刻版）
2. 测完分享结果 → 想看别人怎么玩 → 产生内容消费
3. SBTI测完了 → 还想测类似的 → 找下一个测评

**核心定位**：让用户**找到 → 测完 → 分享 → 再找下一个**的闭环

## 四、功能规划

### P0 · 必须有（v1.0上线）

#### 4.1 测评收录页
- 测评名称 + 图标 + 简介（50字内）
- 标签：SBTI官方 / 复刻版 / 同类推荐 / 隐藏人格等
- 一键跳转直达测评页面
- 收录时间 + 热度标识（🔥 hottest / 🆕 new）

#### 4.2 分类筛选
- 按类型：SBTI / MBTI复刻 / 趣味人格 / 职业测评 / 情感测评
- 按热度排序：最新收录 / 最多使用 / 评分最高
- 按标签筛：隐藏人格 / 免费 / 中文 / 英文

#### 4.3 测评详情卡
- 名称、封面图、简介
- 适合人群（学生/职场/单身等）
- 题目数量、预估耗时
- 用户评分（5星制）
- 一句话评价（精选用户反馈）
- 直达链接（显眼的"立即测试"按钮）

#### 4.4 SEO优化
- 独立URL：/tools/sbti-official
- Meta标签、Open Graph
- 文章页：每个测评配一篇简短测评攻略/解读

### P1 · v1.1迭代

#### 4.5 用户反馈区
- 用户可提交对测评的评价（1-5星 + 一句话）
- 显示"XX人觉得准"

#### 4.6 热门测评榜单
- 日榜 / 周榜 / 总榜
- 标注上升最快的测评

#### 4.7 测评结果分享页
- 用户粘贴结果截图，可生成带站logo的分享卡
- 引导关注公众号（涨粉钩子）

### P2 · v1.2迭代

#### 4.8 测评合集专题
- 「SBTI全版本对比」「测完SBTI还能测这些」
- 专题页SEO权重更高

#### 4.9 开发者入口
- 开放API，让测评工具主动上报数据
- 付费置顶/推荐功能

#### 4.10 公众号/社群引导
- 测完引导关注公众号，回复"SBTI"领隐藏人格攻略
- 私域沉淀，后续推送其他测评

## 六、技术方案

### 方案选型
| 选项 | 方案 | 适合 |
|------|------|------|
| **A（推荐）** | Next.js + Tailwind + Supabase | 快速迭代、SEO友好 |
| **B** | Docsify静态页 | 极简、SEO弱、不推荐 |
| **C** | Nuxt.js + NodeCMS | 复杂、数据驱动 |

**推荐方案A**，2周可上线MVP。

### 数据模型

```typescript
// 测评工具
interface Tool {
  id: string
  name: string
  slug: string          // URL别名
  icon: string          // 图标URL
  coverImage: string
  description: string   // 50字简介
  longDescription?: string  // 详细说明
  tags: string[]        // 标签
  category: string      // 分类
  questionCount: number
  duration: number      // 分钟
  audience: string[]    // 适合人群
  url: string           // 测评跳转链接
  isSBTI: boolean       // 是否SBTI相关
  isHidden: boolean      // 是否隐藏人格
  score: number         // 平均分 1-5
  reviewCount: number   // 评价数
  viewCount: number     // 浏览数
  status: 'pending' | 'approved' | 'rejected'
  createdAt: Date
}

// 用户评价
interface Review {
  id: string
  toolId: string
  score: number
  comment: string
  createdAt: Date
}
```

### 运营数据
- Google Analytics 页面PV/UV
- UTM追踪每个测评的点击导出量
- 广告点击/展示数据（AdSense后台）
